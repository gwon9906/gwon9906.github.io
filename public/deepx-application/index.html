자기소개

1. 왜 DEEPX인가

DEEPX가 지향하는 온디바이스 AI와 저전력·고성능 NPU 방향성은 제가 해 온 프로젝트 경험과 가장 맞닿아 있습니다. 저는 좋은 모델을 만드는 것에서 끝나지 않고, 제약이 큰 환경에서 어떻게 더 가볍게 만들고 실제로 동작하게 만들 수 있는지를 중요하게 봐 왔습니다. 센서 압축 프로젝트에서는 Autoencoder를 ONNX로 변환해 ONNX Runtime에서 추론 결과가 일치하는지 검증했고, LoRa 프로젝트에서는 payload를 줄이는 방식이 실제 고손실 무선 환경에서 전송 성공률 개선으로 이어지는지 반복 필드 테스트로 확인했습니다. DEEPX에서도 모델을 실제 디바이스와 런타임 제약에 맞게 다듬고 검증하는 SW 엔지니어로 기여하고 싶습니다.

2. 저를 가장 잘 설명하는 키워드: 검증

저는 성능 수치만으로 결론을 내리기보다, 왜 그런 결과가 나왔는지 같은 조건에서 비교하고 확인하는 방식을 선호합니다. 산업용 유량 예측에서는 데이터의 유효 자릿수와 시계열 불연속성을 먼저 점검해 전처리 기준을 조정했고, 기본 LSTM과 Encoder-LSTM을 비교해 MAPE를 개선했습니다. 센서 압축 프로젝트에서는 복원 오차만 보지 않고 ONNX 변환 후 Runtime 추론 결과가 원 모델과 allclose로 일치하는지 검증했습니다. LoRa 프로젝트에서도 실험실 결과에 머무르지 않고 NLOS 환경에서 반복 전송을 통해 실제 개선 여부를 확인했습니다. 이처럼 저는 모델링, 경량화, 런타임 검증을 하나의 흐름으로 묶어 설명할 수 있습니다.

주요 경험

센서 데이터 경량 압축 및 ONNX 추론 검증

개인 프로젝트 · 2026.03 - 진행 중

역할: 데이터 구성, Autoencoder 학습, ONNX export, Runtime 검증
  • Intel Lab 센서 데이터를 기반으로 8차원 입력을 구성하고 Autoencoder와 BAM을 비교해 복원 특성 검토
  • PyTorch 모델을 ONNX로 export한 뒤 ONNX Runtime에서 encoder 및 full model 추론 결과를 검증
  • 동적 batch 입력과 재구성 출력의 수치 일치 여부를 확인해 배포 가능한 형태로 모델 구조 정리
  • 의미: 모델 학습뿐 아니라 경량 표현, 추론 호환성, 배포 전 검증까지 연결한 경험

LoRa 저신호 환경 통신 개선

4인 팀 프로젝트 · 2025.03 - 2025.06

역할: BAM 기반 압축 구조 구현, 송수신 파이프라인 구성, 필드 테스트
  • payload 증가가 전송 성공률 저하로 이어지는지 검증하기 위해 GPS·IMU 센서 데이터 기반 압축 파이프라인 구성
  • Raspberry Pi 기반 Linux 환경에서 RAW 32B와 압축 20B payload를 동일 장비·동일 설정으로 반복 비교
  • 약 2.0 - 2.6km NLOS 구간에서 100회 전송 실험을 수행해 실제 제약 환경의 개선 여부 확인
  • 결과: 성공 패킷 수 29 → 33 / 100 tx, PDR 33%, MSE 0.003676

산업용 밸브 정상 유량 예측

개인 연구 · 2024.07 - 2024.12

역할: 데이터 분석, 전처리 기준 정리, Encoder-LSTM 설계, 실험 검증
  • 정상 상태 기준선을 만들기 위해 실제 운용 밸브 시계열 데이터를 사용해 예측 문제 정의
  • 개도율 변화에 따른 불연속성, 제한된 측정 정밀도, 이상치 특성을 반영해 전처리 기준 재설정
  • 기본 LSTM과 Encoder-LSTM을 비교하고 Huber Loss를 적용해 이상치 민감도를 낮춘 구조 검증
  • 결과: Loss 4.0376e-05 / MAE 0.003941 / MAPE 0.183501
