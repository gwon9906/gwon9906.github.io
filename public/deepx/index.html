자기소개

저는 새로운 문제를 마주했을 때 곧바로 모델이나 기법부터 고르기보다, 먼저 데이터가 어떤 조건에서 수집되었는지와 어떤 제약이 성능에 직접 영향을 주는지부터 정리합니다. 산업용 밸브 유량 예측 프로젝트에서는 개도율 0 구간의 불연속성과 제한된 측정 정밀도를 반영해 전처리 기준을 다시 세웠고, LoRa 프로젝트에서는 payload 크기와 전송 성공률의 관계를 실제 현장에서 반복 측정하며 개선 효과를 확인했습니다. 이 과정에서 성능 수치만 좋은 결과보다, 왜 좋아졌는지 설명할 수 있고 다시 검증할 수 있는 결과를 만드는 것이 중요하다는 기준을 갖게 되었습니다. 저는 이러한 방식으로 DEEPX에서도 하드웨어 제약을 고려한 온디바이스 AI SW를 구현하고 검증 가능한 형태로 발전시키고 싶습니다.

지원동기

DEEPX는 전력과 발열이라는 실제 제약을 해결하며 온디바이스 AI 반도체를 제품 수준으로 만들어 가는 회사라는 점에서 매력적이었습니다. 저는 학부 연구 과정에서 제한된 자원과 열악한 통신 환경 속에서도 동작 가능한 구조를 설계하고, 동일 조건 비교와 현장 검증을 통해 개선 여부를 확인하는 경험을 쌓았습니다. 이러한 경험은 추론 엔진, 전처리·후처리, 검증 자동화처럼 성능뿐 아니라 시스템 제약을 함께 고려해야 하는 SW Engineering 업무와 잘 맞닿아 있다고 생각합니다. DEEPX에서 저는 모델을 단순히 실행하는 수준을 넘어, 제약 환경에서도 신뢰할 수 있는 입력 처리·검증·운영 흐름을 만드는 엔지니어로 성장하고 싶습니다.

프로젝트 기술서

산업용 밸브 정상 유량 예측

개인 연구 · 2024.07 - 2024.12

역할: 데이터 분석, 전처리 기준 수립, Encoder-LSTM 설계, 실험 검증
  • 정상 상태 기준선을 만들기 위해 실제 운용 밸브 시계열 데이터를 사용해 유량 예측 문제를 정의했습니다.
  • 개도율 변화에 따른 불연속성, 제한된 측정 정밀도, 이상치 특성을 반영해 전처리 기준을 조정했습니다.
  • 기본 LSTM과 Encoder-LSTM을 비교하고 Huber Loss를 적용해 이상치 민감도를 낮춘 구조를 검증했습니다.
  • 결과: Loss 4.0376e-05 / MAE 0.003941 / MAPE 0.183501

LoRa 저신호 환경 통신 개선

4인 팀 프로젝트 · 2025.03 - 2025.06

역할: BAM 기반 압축 구조 구현, 송수신 파이프라인 구성, 필드 테스트
  • payload 증가가 전송 성공률 저하로 이어지는지 검증하기 위해 GPS·IMU 센서 데이터 기반 압축·복원 파이프라인을 구성했습니다.
  • Raspberry Pi 기반 Linux 환경에서 RAW 32B와 압축 20B payload를 동일 장비·동일 설정으로 반복 비교했습니다.
  • 약 2.0 - 2.6km NLOS 구간에서 100회 전송 실험을 수행해 실제 제약 환경의 개선 여부를 확인했습니다.
  • 결과: 성공 패킷 수 29 → 33 / 100 tx, PDR 33%, MSE 0.003676
